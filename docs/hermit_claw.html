Slide 1 · Platform

CheMLFlow Today — High-level Platform Overview

CheMLFlow is the execution + provenance engine: it turns datasets into reproducible ML pipeline runs with strict validation, deterministic artifacts, and guardrails for fair comparison.

Inputs
  • Local CSV (unknown schemas, real-world messiness)
  • Benchmark sources (e.g., ChEMBL / TDC style flows)
  • One YAML config describing pipeline nodes + params

Goal: make “CSV → valid run” achievable without platform-specific tribal knowledge.

Engine
  • DOE generator expands a spec into runnable case configs
  • Strict validation (schema + node compatibility)
  • Pipeline runtime executes nodes: curate → split → featurize → train → eval

Guardrails are first-class: invalid combinations are rejected before runtime whenever possible.

Outputs
  • Audit trail: run_config.yaml, split_meta.json, metrics
  • Lifecycle status: run_status.json (running/failed/success)
  • DOE manifest: summary.json + manifest.jsonl with skip reasons

Everything needed to trace: case → config hash → run dir → artifacts → metrics.

Core contracts (what stays stable)

# 1) Generate DOE (emits summary + manifest) python scripts/generate_doe.py --doe # 2) Run a single case (runtime reads CHEMLFLOW_CONFIG) CHEMLFLOW_CONFIG= python main.py

Fair-comparison defaults

  • Consistent split policy across cases (or explicit grouping)
  • No “win by dropping rows” (coverage/disjoint rules)
  • Reproducible paths + deterministic run artifacts
CheMLFlow is the “science-grade runner + provenance layer.” The agent layer should rely on these stable artifacts and reason codes.
Slide 2 · Agent

HermitClaw — The Agent Layer (Not Just an Orchestrator)

HermitClaw is the decision + automation layer that makes CheMLFlow usable for real users: it translates intent into valid runs, enforces scientific guardrails, and turns results into defensible next actions.

What it adds
  • Spec translation: objective + CSV → DOE spec + runnable configs
  • Guardrails: block invalid/biased comparisons before execution
  • Triage: map failures to fixes using structured artifacts (not stack traces)
  • Selection: choose “best config” using reproducible, auditable evidence
  • Next-step planning: propose what to run next under a budget
Why this matters for labs
  • New users don’t know the right model family/split/featurization.
  • Bad comparisons waste time (and can mislead decisions).
  • Clinical collaborators need traceability: “what ran, why, and can we reproduce it?”

HermitClaw turns CheMLFlow from “powerful engine” into a guided, reliable workflow.

What it is (practical definition)

HermitClaw is a wrapper that can run allowlisted commands, read artifacts, and make structured decisions. It becomes valuable when it:

  • decides what to run (bounded DOE, correct defaults)
  • knows what not to run (invalid combos, leakage risks)
  • explains outcomes with links to evidence artifacts

How it connects to “cure cancer”

It accelerates the measure → model → decide loop by reducing setup friction, preventing invalid science, and making results reproducible across teams. That shortens iteration cycles in hit discovery, optimization, and assay modeling workflows.

It speeds up the loop you can control
CheMLFlow runs experiments; HermitClaw makes the experiment loop usable, defensible, and repeatable for non-experts.
Slide 3 · Usage

How HermitClaw Will Be Used with CheMLFlow

End-to-end: intake a CSV → generate bounded DOE → run valid cases → monitor lifecycle → aggregate results → recommend next actions (with an audit trail).

Agent flow (deterministic, auditable)

  1. Read CSV header; resolve SMILES + target columns (or ask user).
  2. Create DOE spec with a budget; if grid > 10k, require constraints.max_cases.
  3. Run DOE generation; read summary.json + manifest.jsonl (skip reasons are authoritative).
  4. Run only valid cases; poll per-case run_status.json until success/failure.
  5. Aggregate metrics across folds/repeats; rank configs with tie-breakers + uncertainty.
  6. Emit recommendation: best config + “why” + evidence links; propose the next DOE refinement.

Command + artifact contract

# Generate DOE python scripts/generate_doe.py --doe # Key DOE outputs /summary.json /manifest.jsonl /doe_spec.input.yaml # Case execution CHEMLFLOW_CONFIG=/__...yaml python main.py # Per-case lifecycle (namespaced rerun isolation) ///run_status.json ///run_config.yaml
Key idea

HermitClaw doesn’t guess. It reads structured outputs (manifest + status + metrics) and responds deterministically.
